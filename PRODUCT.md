# Product

## Register

product

## Platform

web

## Users

Any .NET MAUI developer — indie makers shipping 1-2 apps and agency teams managing many. They live in the terminal, juggle multiple MAUI projects across iOS and Android, and need to move fast: bump versions, pick a device, build, archive, deploy. The context is always a release workflow, not exploration.

## Product Purpose

One interactive interface for version management, device selection, build, and deployment across iOS and Android MAUI apps. Eliminates the multi-file, multi-tool, multi-config drudgery: no more editing Info.plist, AndroidManifest.xml, and .csproj by hand; no more hunting UDIDs; no more remembering the right `dotnet publish` flags. MAUI Forge replaces repetitive manual steps with a focused, stateful terminal UI (and an optional web dashboard) that remembers everything per project.

Success is a developer who can open a terminal and go from "I need to release" to "it's building" in seconds, without switching contexts, without looking up flags, and without configuration files.

## Positioning

Your MAUI release copilot.

## Brand Personality

Precise, fast, confident. The tool should feel like a surgical instrument: it does one category of work, deeply, and gets out of the way. No splash screens, no animation, no ceremony. Every action shows a clear preview before it executes. State is always visible — version sync, git status, device selection — so the developer never wonders what's happening.

## Anti-references

Bloated enterprise UIs. Must not resemble Azure DevOps, Jenkins, or heavyweight CI dashboards. No progress-bar jungles, no nested configuration trees, no "wizards" that ask more questions than they answer. Avoid the generic-task-runner trap — this is MAUI-specific and opinionated.

## Design Principles

1. **Get out of the way.** The tool disappears into the task. No splash, no onboarding tour, no animation. The developer is already in flow; the fastest path from open to action wins.

2. **Precision by default.** Every state-changing action shows a before/after preview before it commits. The developer confirms what they're about to do, every time.

3. **Confidence through clarity.** All relevant state is visible at once: versions per platform, sync status, git status, device selection, build config. No hidden surprises, no drill-downs to discover the current state.

4. **Opinionated but not rigid.** Sensible defaults for MAUI workflows (net9.0-ios, net9.0-android, Release/Debug) that work immediately, but every setting is overridable and remembered per project.

5. **Terminal-native, dashboard-augmented.** The terminal UI is the primary surface — keyboard-driven, fast, Spectre.Console rendered. The web dashboard is a richer secondary view for inspection and remote use, never a prerequisite.
